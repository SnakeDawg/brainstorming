# Roadmaps

Now / Next / Later roadmaps. Every item links **back** to the strategy pillar
it serves and **forward** to the issue(s) implementing it.

Markdown roadmaps are the durable, reviewable record. For an interactive
timeline view (drag items between iterations, group by status), use a GitHub
**Projects** board alongside these — setup steps are in
[`POC-GUIDE.md`](https://github.com/SnakeDawg/brainstorming/blob/main/POC-GUIDE.md).

## Adding to a roadmap

- Edit the roadmap table directly, or
- Open a **Feature brief** issue from the
  [issue forms](https://github.com/SnakeDawg/brainstorming/issues/new/choose) —
  when it's closed with the `kb` label, the automation appends a row to the
  roadmap for you.

New roadmap? Copy [`_template.md`](_template.md) and add it to the table above.
