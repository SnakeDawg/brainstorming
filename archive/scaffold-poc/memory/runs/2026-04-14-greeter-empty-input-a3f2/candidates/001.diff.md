# Candidate 001 — trim whitespace and guard empty input

## Rationale

The baseline run showed t2 and t3 failing because
`format-greeting/SKILL.md` had no explicit edge-case handling for empty
inputs or whitespace-padded names. The minimum viable fix is to
document those two edge cases directly in the skill's instructions
section so that any LLM executing the skill will apply them.

This diff does not change the happy-path behavior (t1 "Ada" still
produces "Hello, Ada! Welcome."), so no regression is expected on t1.

Only the Edge Cases subsection of the skill body is modified; the
frontmatter and the rest of the content are untouched.
